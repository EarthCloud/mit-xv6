## 1. 进程栈的基本结构

完成该实验需要大致理解进程栈的结构，依据Lec 5中给出的参考文档：
1. 在标准 RISC-V 调用约定中，栈向**下**生长，且栈指针 (`sp`) 始终保持 **16 字节对齐** 。
2. s0/fp寄存器始终指向栈底（高地址），该寄存器由被调用者保存。
3. 过程发生调用时，栈底依次存储ra（返回地址），fp（栈底指针）以及定义的局部变量。特别地，如果该函数为叶子函数的话，不需要存储ra，因其不会调用其他函数，其不会被覆盖，此时，fp贴近栈底。

实验中给出的被调用过程实例如下：

```c
int

f(int x)

{

   e:   1141                    addi    sp,sp,-16

  10:   e422                    sd  s0,8(sp)

  12:   0800                    addi    s0,sp,16
  // inline function g(x)
  return x+3;

}

  14:   250d                    addiw   a0,a0,3

  16:   6422                    ld  s0,8(sp)

  18:   0141                    addi    sp,sp,16

  1a:   8082                    ret
```

RISC-V精简指令集不支持x86那样的压栈操作。于是，进入函数调用时，我们必须显式地生长栈指针：

```c
   e:   1141                    addi    sp,sp,-16

  10:   e422                    sd  s0,8(sp)

  12:   0800                    addi    s0,sp,16
```

`addi`指令将立即数-16加到sp寄存器上，因此进程栈向下生长，然后，它在偏移量sp+8，即贴近栈底的位置存储fp（该函数为叶子函数，g(x)被内联了）。但是为了16字节对齐，sp仍然生长16。

在backtrace中，我们需要读取栈上存储的fp，来进入上个函数的栈底，再根据上个函数栈底存储的ra，输出对应的pc，以此类推直达进程栈底（幸运的是，xv6的进程栈只有一页大，我们可以轻松计算出它的位置）。

## 2. 实现

首先添加读取fp的内联汇编（riscv.h）：

```c

static inline uint64

r_fp()

{

  uint64 x;

  asm volatile("mv %0, s0" : "=r" (x) );

  return x;

}
```

注册backtrace系统调用存根的过程略，下一步，在printf.c中实现backtrace：

```c
// in printf.c:136
void

backtrace()

{

  printf("backtrace:\n");

  uint64  fp = r_fp();

  uint64 *p = (uint64 *)fp, *stack_top = (uint64 *)PGROUNDUP(fp);

  while(p < stack_top) {

    printf("%p\n", *(p - 1));

    uint64 *next_p = (uint64 *)(*(p - 2));

    if(next_p <= p) {

      break;

    }

    p = next_p;

  }

}
```

有大量的强转，阅读时务必注意。
