# lab2 syscall

由于syscall trace的测试会用到README文件，我新建了一个markdown来记录实验内容。

## trace

在这个实验中，我实现了一个简单的系统调用`trace`，它可以追踪指定进程的系统调用并输出相关信息。具体实现过程参考[trace](./notes/1%20system%20call%20tracing.md)。

你需要完成：
1. 理解如何在xv6中添加新的系统调用。
2. 深入理解xv6系统调用的生命周期。
3. 实现`trace`系统调用，能够接受一个整数参数作为掩码，表示要追踪的系统调用类型。在这个过程中，进一步理解二进制压缩的奇妙。
4. 了解内核的系统调用如何获取用户程序的参数。
5. 在学习内核分发系统过程中，理解函数指针数组的使用。

## sysinfo

在这个实验中，我实现了一个新的系统调用`sysinfo`，它可以返回系统的一些基本信息，如内存使用情况和进程数量。具体实现过程参考[sysinfo](./notes/2%20sysinfo.md)。

你需要完成：
1. 完全理解嵌入式链表的作用机制（虽然可能不太用得到）
2. 学会在内核代码中定位关键数据结构
3. 理解如何安全地访问和修改内核数据结构，理解如何使用锁机制
